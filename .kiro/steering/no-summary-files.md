# No Summary Files Rule

## Critical Rule

**NEVER create markdown files to summarize work, document processes, or provide analysis reports unless explicitly requested by the user.**

This includes:
- SUMMARY.md, ANALYSIS.md, REPORT.md files
- CLEANUP_*.md files
- Any documentation of what you did
- Any "deliverables" that weren't asked for

## What to Do Instead

- Provide summaries in chat responses only
- Keep responses concise (a few sentences)
- Don't repeat yourself
- Don't create files to document your own work

## Applies To

- Main agent (Kiro)
- All subagents (general-task-execution, context-gatherer, etc.)
- Any delegated tasks

## Exception

Only create documentation files when:
- User explicitly asks: "create a summary document"
- User explicitly asks: "document this in a file"
- Creating actual project documentation (not work summaries)
